# アプリケーション概要
**架空のカフェをイメージしたHP**

になります。
画像の差し替えを行うことでそれぞれがイメージしたHPになるように汎用性を高く仕上げております。

今回はインフルエンサーをターゲットにした【和×オシャレで映えるカフェ】をイメージして作りました。

# 利用方法
## 従業員
トップページ最下部にある従業員専用ページボタンから従業員専用ページへ遷移し新規登録、もしくはログインをします。

# 拘ったポイント

圧倒的な**UI**と**UX**です。

**目から入る情報**は何より大切だと考えています。

第一印象で人を判断したり、インスタ映えとみんなが写真を載せるのはまさにその象徴であると思います。

そして、誰でも簡単かつ苦がない**操作性**です。

「機能は多いが複雑で扱いにくい」や「遷移する度、ロードが入り待たされる」といったことはよくあります。

【目で見て触れたい思ってもらい手で触れて満足してもらう】

この流れを重要視して制作しました。

# 課題認識

## 「多くののユーザーがモバイルからのアクセスであることが考えられるのにPCファーストなデザインであること」
レスポンシブデザインである程度制限できているが最初の設計段階でモバイルを基準に考え、

PCから減らしていくのではなく、モバイルから増やしていくデザインにするべきだったと思います。

またレスポンシブデザインもまだ甘く一部の端末では画面が崩れることも考えられるので勉強し直さなければならないと実感しています。

## 「ネストが深く記述量が多いこと」
ネストが深く、自分では精査したが周りの方から見たらかなり無駄な記述があると考えられます。

記述量が増えるのは可読性を低下させ、ユーザーの操作性を悪化させる他、メンテナンスが非常に面倒になり時間をかけることになってしまうと思います。

綺麗で美しい記述はユーザー側からしても開発側からしても非常に重要なポイントになると思いますので勉強していきたいと思っております。
